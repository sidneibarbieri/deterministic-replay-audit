# Reviewer guide

**Estimated time: 1 minute to trust, ~10 minutes to fully reproduce.**

This artifact is the reproducibility package for *Auditing AI Investment
Recommendations as Executable Actions*. You do not need to read any code. Every
reported quantitative result and empirical figure is regenerated from frozen
inputs. The trust check is offline; full host reproduction requires the usual
Python and frontend dependencies to be installed or cached.

## In 60 seconds

| Question | Answer |
|---|---|
| What does this artifact do? | Audits an AI advisor's recommendation as an executable financial action (valid, stable, grounded) **before** any return is computed. |
| Main contribution | Admissibility is independent: declaring the contract lifts both frontier models to 0.94 validity, and computed arithmetic removes the remaining numeric failures. |
| How do I reproduce? | `make review` (trust check) or `make all` (full). |
| How do I verify it reproduced? | `make review` re-derives the paper's headline numbers from the frozen data and prints PASS for each. |
| Where are the paper's figures? | `paper/figures/*.pdf`, regenerated by the commands in the matrix below. |

## Three commands

```bash
make review         # ~1 min: re-derives the paper's numbers, no network, no API keys
make all            # ~10 min after setup: tests and regenerates empirical outputs
open paper/data/review_dashboard.html   # per-model audit dashboard (offline HTML)
```

`make review` prints, for each of the six frontier runs, the validity rate next
to the value claimed in the paper, plus `truncated=0` and prompt-provenance
checks. All 432 frozen prompts must match both their saved SHA-256 and the
current prompt builder, and every raw response must reparse to its stored
structured output. If every line says PASS, the headline experiment reproduced.

## Run the models yourself (optional)

The frozen runs ship with the artifact, so no keys are needed to reproduce. To
re-collect live model output, add your own keys to `.env` (see `.env.example`)
and run `make advisor-experiment`. The collector retries only transient errors,
records a truncation flag, and writes a hashed manifest.

## For reviewers who are not software engineers

You only need Docker (or Python 3.11). Open a terminal in this folder and type
`make review`. Read the lines it prints: each `[PASS]` means a number in the
paper was recomputed from the saved data and matched. The trust check does not
contact the internet, and no money or trade is ever involved. The optional dashboard
(`paper/data/review_dashboard.html`) opens in any browser and shows, per model
and prompt, how often the advice was valid and which rules it broke.
